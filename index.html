3.2 Data Stored Locally on Your Device Only

The following data is stored exclusively in Chrome's local storage on your device and is never transmitted to our servers:

  • Scan history (configurable: 10–100 entries)
  • Sender memory (which senders you've scanned)
  • Whitelist and blacklist of domains you define
  • Scan statistics and weekly reports
  • Your settings and preferences
  • Community Trust votes you have cast (domain + vote only)

3.3 Native Agent — Local Processing

When using the Native Agent for file scanning or Windows Sandbox features:

  • The Agent downloads attachment files temporarily to your device at C:\Users\[you]\Desktop\SecureMailAgent\temp_[filename]
  • Files are automatically deleted after scanning or download
  • The Agent creates temporary script files (.wsb, .cmd) for Sandbox operation — deleted automatically after use
  • A debug log is written locally at SecureMailAgent\debug.log — contains email subjects and senders for troubleshooting. This file stays on your device only.
  • The Agent reads Gmail headers via Gmail API using OAuth credentials you authorized — scope: gmail.readonly

3.4 Community Trust Data

When you vote on a domain (👍 Safe / 👎 Dangerous):

  • Only the domain name and your risk rating are sent to our database (Supabase)
  • Your IP address is used server-side to prevent duplicate votes — it is not stored in the database
  • No personal identifier is attached to your vote
  • You can disable Community Trust voting entirely in Settings

4. Google Gmail API — Compliance Statement Gmail API ToS

Shomer's use of the Gmail API complies with Google's API Services User Data Policy, including the Limited Use requirements:

  • Limited Use: Gmail data is used solely to provide the email security scanning service. It is not used for advertising, user profiling, or any purpose unrelated to security analysis.
  • No storage: Gmail message content is never stored on our servers beyond the in-memory cache (max 1 hour) required for technical processing.
  • No sharing: Gmail data is not transferred to third parties except as necessary for scanning (VirusTotal for file hashes, Groq AI for sanitized text analysis). These services receive only the minimum data necessary.
  • No human access: No Shomer employee or developer reads your email content. All processing is automated.
  • Scope justification: The gmail.readonly scope is required to read email authentication headers (SPF, DKIM) and download attachments for scanning — features that cannot function with a narrower scope.
  • Revocation: You can revoke Shomer's Gmail access at any time at myaccount.google.com/permissions.

5. Automatic PII Sanitization

Before any email body text leaves your device, Shomer automatically replaces the following with placeholders:

  • Credit card numbers → [CARD]
  • Israeli phone numbers → [PHONE]
  • Email addresses → [EMAIL]@domain.com (domain preserved for analysis)
  • Israeli ID numbers (9 digits) → [ID]
  • IP addresses → [IP]
  • Passwords (e.g. "password: xxx") → [REDACTED]

This sanitization is performed client-side before any data is transmitted.
